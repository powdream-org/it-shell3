# Implementation Learnings

Cross-cutting lessons from implementation work. Updated as new modules are
implemented.

---

## Zig Toolchain

| #  | Lesson                                                      | Origin          | Impact                                                                                                                                                                                                                                                                                                                        |
| -- | ----------------------------------------------------------- | --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Z1 | **Vendored C must compile with ReleaseSafe**                | libitshell3-ime | Zig applies UBSan instrumentation to C code in Debug mode. This conflicts with ptrace-based tools (kcov, debuggers), causing segfaults. Compile vendored C libraries with `.ReleaseSafe` to disable UBSan while keeping safety checks. Applies to any module wrapping vendored C.                                             |
| Z2 | **Zig linker Mach-O blocks instrumented coverage on macOS** | libitshell3-ime | Zig's self-hosted linker leaves insufficient headroom between load commands and `__text` section offset. DWARF-dependent tools (nm, dsymutil, kcov) fail on the resulting binary. [ziglang/zig#31428](https://codeberg.org/ziglang/zig/issues/31428). Monitor upstream for fix before enabling coverage on other Zig modules. |
| Z3 | **`= undefined` is real UB**                                | libitshell3-ime | Unlike C's indeterminate value, Zig's `= undefined` is genuine UB that can cause crashes under instrumentation. Use `@splat(0)` for buffers that may be read before full initialization.                                                                                                                                      |

## Testing

| #   | Lesson                                                              | Origin          | Impact                                                                                                                                                                                                                                                                                                                    |
| --- | ------------------------------------------------------------------- | --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| T1  | **Scenario-matrix tests are a viable coverage substitute**          | libitshell3-ime | When instrumented coverage is blocked by toolchain issues, a named-test-per-spec-path approach provides equivalent confidence. Each spec scenario maps to a uniquely named test, making coverage auditable. Validated with 136 tests across 17 categories.                                                                |
| T2  | **OS vtable interfaces enable deterministic testing from day one**  | libitshell3     | PTY, kqueue, sockets, and signals behind vtable interfaces allow all server logic to be tested with mocks. Real OS resources only in integration tests. Prevented test hangs that plagued early PTY/socket tests.                                                                                                         |
| T3  | **Real kqueue tests use pipes, not sockets or PTYs**                | libitshell3     | Pipe pairs provide deterministic kqueue events without fork, network, or blocking accept. Each test creates its own kqueue fd + pipe — fully isolated. Verify actual data after kqueue reports readiness.                                                                                                                 |
| T4  | **Inline large buffers cause stack overflow in static arrays**      | libitshell3     | Structs with large inline buffers (32KB+) stored in static arrays (e.g., 64-slot `ClientEntry` with 128KB buffer = 8MB) cause stack overflow. Use lazy heap allocation (`page_allocator`) or redesign to avoid per-slot buffers.                                                                                          |
| T5  | **QA tests must derive from the spec, not from the implementation** | libitshell3     | Tests that verify "the code does what the code does" are tautological. QA must read the spec independently and test whether the implementation matches the spec's requirements. A copy-based delivery path that works correctly is still a spec violation if the spec requires zero-copy iovecs.                          |
| T6  | **`refAllDecls` does not cross named module boundaries**            | libitshell3     | Named modules (`@import("module_name")`) are separate compilation units. `refAllDecls(@This())` only discovers tests within the current unit. Per-module test steps in build.zig required — each named module gets its own `addTest`.                                                                                     |
| T7  | **Concurrent agents sharing workspace can lose files**              | libitshell3     | Implementer and QA running in the same workspace can overwrite each other's files. QA's 102 spec tests were silently lost. Sequence agents or use worktree isolation. Verify file existence after agent completion.                                                                                                       |
| T8  | **Clean zig-cache before kcov to ensure fresh binaries**            | libitshell3     | Cached builds don't update binary timestamps. `find -newer` fails to find test binaries. Remove `.zig-cache` before the coverage build to guarantee all binaries are fresh.                                                                                                                                               |
| T9  | **Linux ucred struct missing from Zig std**                         | libitshell3     | SO_PEERCRED needs ucred. Zig 0.15.2 omits it. Define: `extern struct { pid: i32, uid: u32, gid: u32 }`. Found in Docker.                                                                                                                                                                                                  |
| T10 | **timerfd_create arg order**                                        | libitshell3     | Takes (clockid, TFD) not (TFD.TIMER, clockid). TFD has CLOEXEC; TFD.TIMER is for timerfd_settime. Found in Docker.                                                                                                                                                                                                        |
| T11 | **Spec says "integer arithmetic" — enforce it literally**           | libitshell3     | When the spec mandates "No floating-point is involved at any step," implementers may still use f32 conversion for convenience (e.g., `@floatFromInt(ratio) / RATIO_SCALE`). Owner caught this in review. All geometry (split_tree, navigation, notification_builder) must use `width * ratio / 10000` integer arithmetic. |

## ghostty Integration

| #  | Lesson                                                                   | Origin      | Impact                                                                                                                                                                                                                             |
| -- | ------------------------------------------------------------------------ | ----------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| G1 | **ghostty `-patch` tags break build system version detection**           | libitshell3 | ghostty's `Config.zig` panics on tags not matching `vX.Y.Z`. Use `-Dversion-string=X.Y.Z` to bypass git tag detection when pinning to fork tags with suffixes.                                                                     |
| G2 | **`lib_vt.zig` is the headless entry point — not all APIs are exported** | libitshell3 | Mouse encoding (`mouse_encode.zig`) exists in ghostty source but is NOT exported via `lib_vt.zig`. Only Terminal, RenderState, key_encode, and paste are available. Daemon must author its own mouse encoder.                      |
| G3 | **ghostty `vtStream()` returns ephemeral parser state**                  | libitshell3 | Creating a new vtStream per `feedTerminal` call loses mid-sequence parser state for split escape sequences. Hold the stream for the session lifetime (persistent vt_stream).                                                       |
| G4 | **ghostty API is unstable across versions**                              | libitshell3 | Pinning to a new version (v1.3.1) broke `MouseEncodeEvent` (removed) and `stream.nextSlice` (became failable). Always test after version bumps. Wrap ghostty via helper functions (not types) to limit blast radius per spec §1.2. |
