# Skill Improvement Proposals

```
draft/vX.Y-rN/retrospective/skill-improvement-proposals.md
```

A **living document** updated throughout the revision cycle whenever the team
leader encounters procedural problems, improper facilitation, or skill gaps. Not
limited to the retrospective step — issues are logged as they occur.

## When to Update

- A step file's instructions are unclear or missing
- An anti-pattern is discovered that isn't documented
- Token waste from unnecessary agent spawns or communication rounds
- A gate condition is ambiguous or insufficient
- The team leader violates an anti-pattern (self-correction)
- The owner identifies a procedural gap during review

## Required Content

Each proposal must include:

| Field        | Description                              |
| ------------ | ---------------------------------------- |
| Number       | Sequential within the cycle              |
| Problem      | What went wrong or what is missing       |
| Evidence     | When it happened and what the impact was |
| Proposed fix | Specific edit to a specific file         |

## Lifecycle

1. **Created** when the first procedural issue is found (any step)
2. **Appended** throughout the cycle as new issues arise
3. **Reviewed and finalized** during Step 9 (Retrospective)
4. **Applied** after owner approval in Step 9
